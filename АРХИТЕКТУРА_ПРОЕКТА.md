# Архитектура проекта (быстрый контекст)

Цель файла — за 1–2 минуты понять **где что лежит** и **куда лезть** при правках (UI/API/БД/агент/отчёты).

---

## Репозиторий (корень)

- **`README.md`**: быстрый старт, что коммитить/не коммитить, общая структура.
- **`run.py`**: запуск API из корня (переключает cwd в `backend/`, запускает `uvicorn`).
- **`package.json`**: корневые npm-скрипты оркестрируют API + фронт (`npm start`, `npm run start:prod`, тесты).
- **`package-lock.json`**: lockfile для корневых dev-зависимостей (concurrently/cross-env/playwright).
- **`pyproject.toml`**: настройки python-проекта/pytest (пути тестов, pythonpath=backend).
- **`start_all.bat`** / **`dev.bat`**: батники под Windows для запуска окружения.
- **`playwright.config.ts`**: e2e-тесты (Playwright).
- **`MIGRATION.md`**: заметки/правила по миграциям SQLite.
- **`CHANGELOG.md`** / **`ROADMAP.md`**: история изменений и план.
- **`glpi.csv`**: пример выгрузки/формата (используется как ориентир для “Заявок”).
- **`cookies.txt`**: вспомогательный файл (обычно для локальных запросов/тестов; в репозиторий в проде не нужен).

Папки:

- **`backend/`**: FastAPI + SQLite + SQLAlchemy (async).
- **`frontend/`**: React + TypeScript + Vite + Tailwind.
- **`agent/`**: агент инвентаризации (Windows, PowerShell/опционально Python), отправка на API.
- **`docs/`**: документация по деплою, контракту агента, ограничениям GLPI-совместимости.
- **`e2e/`**: e2e-тесты/фикстуры (если используются).
- **`scripts/`**: вспомогательные скрипты (если появляются — держим тут).
- **`node_modules/`** и **`frontend/node_modules/`**: генерируются `npm install`, **не** источник архитектуры.

## Backend (`backend/`)

### Вход и запуск

- **`backend/start_server.bat`**: запуск бэкенда (Windows).
- **`backend/open_firewall_port.bat`**: открыть входящий порт в Windows Firewall (для LAN-агентов).
- **`backend/requirements.txt`**: python-зависимости (FastAPI/SQLAlchemy/ldap3/fpdf2 и т.д.).
- **`backend/.env.example`**: шаблон настроек.
- **`backend/.env`**: локальные секреты (пароли/токены) — **не коммитить**.

### Приложение (`backend/app/`)

- **`backend/app/main.py`**: “сборка” FastAPI-приложения.
  - подключает роутеры,
  - делает авто-миграции SQLite через `ALTER TABLE ...` в `lifespan`,
  - в проде раздаёт `frontend/dist` как SPA (fallback на `index.html`).
- **`backend/app/database.py`**: SQLAlchemy engine/session (`AsyncSessionLocal`, `engine`, `Base`).
- **`backend/app/models.py`**: ORM-модели БД (Users, Computers, Service Requests, Templates, Tags и т.д.).
- **`backend/app/schemas.py`**: Pydantic-схемы (DTO) для API.
- **`backend/app/config.py`**: `settings` из `.env` (pydantic-settings).
- **`backend/app/auth.py`**: логин/проверка пароля, сессии/куки, хеширование.
- **`backend/app/ldap_config.py`** и **`backend/app/ldap_auth.py`**: LDAP-конфиг и LDAP-аутентификация/синхронизация.

### Роутеры (`backend/app/routers/`)

Эти файлы — “контроллеры” API (эндпоинты + бизнес-логика):

- **`auth.py`**: вход/выход/`/auth/me`.
- **`users.py`**: управление пользователями + админские LDAP-операции.
- **`settings.py`**: настройки (например LDAP-конфиг).
- **`computers.py`**: ПК: список/детали/редактирование/история/экспорт.
- **`tags.py`**: теги (справочник, CRUD).
- **`dashboard.py`**: агрегаты для дашборда (сводные метрики, топы/распределения).
- **`agent.py`**: приём инвентаризации от агента (`POST /api/v1/agent/inventory`).
- **`agent_tokens.py`**: выпуск/отзыв токенов агентов.
- **`service_requests.py`**: “Заявки” (импорт GLPI CSV, сортировка/фильтры, шаблоны, PDF-экспорт).

## Frontend (`frontend/`)

### Сборка/инструменты

- **`frontend/package.json`**: зависимости UI (React/Router/Chart.js) и скрипты (`dev`, `build`, `test`).
- **`frontend/vite.config.ts`**: Vite-конфиг (в т.ч. прокси `/api` в dev).
- **`frontend/index.html`**: html-шаблон.
- **`frontend/tsconfig*.json`**: TypeScript-конфиг.
- **`frontend/eslint.config.js`**: ESLint.
- **`frontend/README.md`**: заметки по фронту.

### Исходники (`frontend/src/`)

Базовые entrypoints:

- **`frontend/src/main.tsx`**: монтирование React-приложения.
- **`frontend/src/App.tsx`**: маршрутизация (React Router) + охрана роутов (Protected).
- **`frontend/src/AuthContext.tsx`**: контекст авторизации (`me`, `login`, `logout`, user state).
- **`frontend/src/api.ts`**: единая точка API-клиента (fetch, таймауты, типы DTO).
- **`frontend/src/ErrorBoundary.tsx`**: перехват ошибок UI.
- **`frontend/src/index.css`** / **`frontend/src/App.css`**: стили.

Компоненты:

- **`frontend/src/components/icons.tsx`**: набор иконок (используются во всех страницах/сайдбаре).

Pages (страницы/крупные модули):

- **`frontend/src/pages/Layout.tsx`**: общий layout (sidebar/top), навигация по разделам.
- **`frontend/src/pages/LoginPage.tsx`**: экран входа.
- **`frontend/src/pages/DashboardPage.tsx`**: дашборд/графики/сводные метрики.
- **`frontend/src/pages/ComputersPage.tsx`**: “Парк ПК” (таблица, фильтры, модалка деталей и т.д.).
- **`frontend/src/pages/SoftwarePage.tsx`**: каталог ПО/поиск (если включён).
- **`frontend/src/pages/ServiceRequestsPage.tsx`**: “Заявки” (создание/база/шаблоны/статистика в одной странице по роуту).
- **`frontend/src/pages/UsersPage.tsx`**: пользователи (админка).
- **`frontend/src/pages/SettingsLdapPage.tsx`**: LDAP-настройки.
- **`frontend/src/pages/AgentTokensPage.tsx`**: управление токенами агентов.
- **`frontend/src/pages/SettingsTagsPage.tsx`**: управление тегами.

Types:

- **`frontend/src/types/vendor.d.ts`**: временные декларации модулей (например для chart-библиотек), чтобы TS не ругался.

## “Куда дергать меня” (быстрая навигация по задачам)

- **Навигация/меню/сайдбар**: `frontend/src/pages/Layout.tsx`
- **Маршруты страниц**: `frontend/src/App.tsx`
- **Авторизация (UI)**: `frontend/src/pages/LoginPage.tsx`, `frontend/src/AuthContext.tsx`
- **Запросы к API / типы DTO**: `frontend/src/api.ts`
- **“Парк ПК” (UI)**: `frontend/src/pages/ComputersPage.tsx`
- **Дашборд (UI, круги/графики/карточки)**: `frontend/src/pages/DashboardPage.tsx`
- **“Заявки” (весь UX/UI, сортировки, модалка, статистика/графики)**: `frontend/src/pages/ServiceRequestsPage.tsx`

Backend:

- **Подключение роутов/старт приложения/раздача SPA**: `backend/app/main.py`
- **Модели БД**: `backend/app/models.py`
- **Схемы API**: `backend/app/schemas.py`
- **Заявки (импорт/экспорт/PDF/шаблоны)**: `backend/app/routers/service_requests.py`
- **ПК (API)**: `backend/app/routers/computers.py`
- **Дашборд агрегаты**: `backend/app/routers/dashboard.py`
- **Пользователи/LDAP**: `backend/app/routers/users.py`, `backend/app/routers/settings.py`, `backend/app/ldap_*.py`
- **Агент и токены**: `backend/app/routers/agent.py`, `backend/app/routers/agent_tokens.py`

## Примечания по “источникам правды”

- **Данные и формат дат “как в CSV” для заявок**: ориентир `glpi.csv` + логика импорта/вывода в `service_requests.py` и форматирование в `ServiceRequestsPage.tsx`.
- **Прод-режим “один порт”**: фронт собирается в `frontend/dist`, а бэкенд из `backend/app/main.py` начинает раздавать SPA.
